# IT Helpdesk Ticketing Lab

This project simulates a real-world IT helpdesk environment using a Raspberry Pi as the end user. It focuses on ticket creation, troubleshooting, documentation, and resolution of common IT support issues.

---

## Overview
In this lab, I created a simple ticketing system and worked through multiple support scenarios including network issues, login problems, and software installation failures.

Each issue was handled using a structured troubleshooting approach:
1. Identify the problem
2. Diagnose the root cause
3. Apply a solution
4. Document actions and resolution

---

## Lab Scenarios

### Network Connectivity (DNS Issue)
- User unable to access internet
- Diagnosed using `ping` (IP vs domain test)
- Resolved by correcting DNS configuration

### User Login Issue
- User unable to log into system
- Identified account lock status
- Resolved by unlocking account and resetting password

### Software Installation Issue
- Application not available on system
- Verified missing package
- Installed using Linux package manager (`apt`)
